RIJIT CLOUD STORAGE PLATFORM

Secure Cloud Architecture using AWS EC2, EBS, S3, GitHub Actions and Netlify

PROJECT OVERVIEW

This project demonstrates the implementation of a complete cloud storage architecture using Amazon Web Services and DevOps tools. It includes compute, storage, automation, and deployment components.

Objectives

  • Deploy virtual server using Amazon EC2
  • Attach persistent storage using Amazon EBS
  • Store files using Amazon S3
  • Implement CI/CD pipeline using GitHub Actions
  • Deploy website using Netlify

AMAZON EC2 (ELASTIC COMPUTE CLOUD)

Description

Amazon EC2 provides scalable virtual servers in the cloud. It allows hosting applications securely and reliably.

Features

  • Virtual Linux server
  • Secure SSH access
  • Scalable compute power
  • High availability

Use Case

Used to host cloud applications, websites, and backend services.

AMAZON EBS (ELASTIC BLOCK STORAGE)

Description

Amazon EBS provides persistent block storage for EC2 instances. Data remains safe even if instance stops.

Features

  • Persistent storage
  • High performance
  • Secure data storage
  • Easy backup

Use Case

Used for storing databases, logs, and application files.

AMAZON S3 (SIMPLE STORAGE SERVICE)

Description

Amazon S3 provides highly durable object storage for storing files, backups, and media.

Features

  • Highly durable storage
  • Unlimited scalability
  • Secure file storage
  • Global access

Use Case

Used for storing images, videos, backups, and static website files.

CI/CD PIPELINE (GITHUB ACTIONS)

Description

CI/CD automates building, testing, and deploying applications.

Features

  • Automatic build
  • Automatic testing
  • Continuous integration
  • Continuous deployment

Use Case

Used in DevOps to automate software deployment process.

NETLIFY DEPLOYMENT

Description

Netlify hosts websites and automatically deploys from GitHub repository.

Features

  • Automatic deployment
  • Free hosting
  • Fast global delivery
  • Secure HTTPS

SECURITY FEATURES

  • IAM access control
  • Secure SSH access
  • Encrypted storage
  • Access policies

  • EC2 server successfully created
  • EBS storage attached and working
  • S3 storage working
  • CI/CD pipeline successful
  • Website deployed successfully
